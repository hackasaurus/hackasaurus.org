Open Web Widget (ScratcHTML) What are we going to call this thing? The teens at YOUmedia want to call it Hackosaurus ;)

I've been using "Mutability" in my research proposal. Hackasaurus is awesome! Could trend younger, which could be good or bad depending on what age group we want to attract.-Rafi

Our five bullets:

  • We want kids to understand that the web is designed and mutable and that they can make it whatever they want it to be, if they have the right tools (technical and social).
  • Kids like the web but are sometimes scared of "complicated" stuff like coding.
  • That's why we need to teach kids how they can easily personalize the web, and byond that, change the ways things work when they don't line up with what they think the web should be.
  • So, we need fun tools that make HTML and other building blocks of the web understandable.
  • We also need activities that matter to kids and make them want to keep creating.

Thoughts on curriculum:
- It might be good to somehow make transparent not just the web itself, but somehow the process that hackers/coders/developers go through when they are changing the design of something, so that the kids have a model of what "expert practice" looks like which they can then give a try at. This might mean create a space for "walk throughs" or how tos, something of that nature.


QUESTIONS:

1) That's a great idea. How could this course be structured? Would it be in the form of training videos put on a website or webinars (presentations) or something else?
I think that right now we're focused on the tool itself and use cases for it, and as we're doing that, think about how it would be used in practice by educators. Right now the plan is to pilot it in the YouMedia library space in Chicago.(http://youmediachicago.org/)

2) 11-18 youth group is very diverse, there should be different courses for children of different age - an 18 year olds' knowledge of the web is a lot wider that that of 11 y.o. Should we divide them into age subgroups and taylor the courses accordingly?
For sure - one of the challenges for the project I think it to make a tool that has, as Mitch Resnick (one of the designers of Scratch) says "low floors, wide walls and high ceilings", so that it's easy for younger and more novice users to get into, once they're into it and are more intermediate allows for diverse uses, and once they get really good or are potentially coming to the table with some dev skills can still be used to do interesting things. If it's designed that way, it can be taken up differently in different courses and contexts. Flexibility in use is key.

Looks like a good transition path to more detailed courses - p2p university or any other training course focused on a particular area of web expertise.

3) We could also greatly empower community knowledge-sharing by encouraging interested kids to tell a few words about their experiences hacking the web (provide some showcases, asking for or providing help)
Definitely! One thing I'd also really love to see in the tool is something that prompts kids once they've made edits/remixes on a page to justify the design decisions they've made in some way, to get them to think like a designer in that way.

4) Just a suggestion - not sure how it fits into the scheme or already existing plans for the tool: most of the time kids would prefer a game to a text book (and Gamestar Mechanic shows how those two could fused in a fun way to produce a great tool for learning) Would it be appropriate to present the kids with 'I want to gallery' showing them what they could potentially make with the web and eventually linking them with one or more tools to try it. Each tool might be associated with creative-type challenges (an example, some basic notions relevant to the theme they chose and finally a suggestion to hack the web). If they like it and go to the next task (level) it introduces some more 'How things work' and provides greater space for creativity and fun with the web.

Outcome:
Leave kids with an understanding of the web as something that can be changed. That they can produce, not just consume.
